# 剧情拆解模板

## 文档头部信息
```
小说名称：《具体小说名》
小说类型：具体类型分类（悬疑/都市/玄幻/言情等）
```

## 批次标记格式
```
第X批（第X-X章）
```

## 剧情点格式
每个剧情点使用以下固定格式：

```
【剧情X】场景描述，角色行为，事件核心，情绪钩子类型，第X集，状态：未用/已用
```

### 格式说明

#### 【剧情X】编号规则
- 全文连续编号，不按批次重置
- 示例：【剧情1】、【剧情2】、【剧情15】

#### 场景描述要求
- 简洁明确的地点信息
- 关键环境特征
- 示例：派出所审讯室、破旧客厅、学校天台

#### 角色行为描述
- 关键角色在此情节中的核心行为
- 避免细节描述，突出关键动作
- 示例：韩非试镜表演、孟诗邀请吃饭、警察上门询问

#### 事件核心提炼
- 用最简洁的语言概括事件要点
- 突出推动剧情的关键信息
- 示例：激活系统面板、发现冰箱藏尸、被鬼追杀

#### 情绪钩子类型标注
必须从以下类型中选择一种：
- **悬念类**：危机悬念、身份悬念、结果悬念、规则悬念
- **反转类**：身份反转、实力反转、情况反转、认知反转
- **爽点类**：打脸爽点、复仇爽点、成长爽点、收获爽点
- **情感类**：温情陷阱、背叛冲击、牺牲感动、守护决心

#### 分集标注规则
- 标注该剧情点应该出现在第几集
- 考虑单集容量（500-800字）
- 确保每集有完整的情绪起伏
- 示例：第1集、第5集、第12集

#### 状态管理
- **未用**：剧情点尚未被使用创作剧本
- **已用**：剧情点已被使用，避免重复使用

## 完整示例

```
小说名称：《我的治愈系游戏》
小说类型：悬疑/推理/科幻

第1批（第1-6章）

【剧情1】出租屋，韩非戴上游戏头盔，被不明尖刺刺入大脑，危机悬念，第1集，状态：未用
【剧情2】游戏内客厅，韩非苏醒发现环境破旧诡异，反差悬念，第1集，状态：未用
【剧情3】楼道，邻居孟诗阿婆邀请韩非吃饺子，温情陷阱，第1集，状态：未用

第2批（第7-12章）

【剧情16】派出所，韩非接受审讯并结识刑警，得知古董店老板死亡，身份悬念，第7集，状态：未用
```

## 质量要求

1. **信息完整**：每个剧情点必须包含所有6个要素
2. **格式统一**：严格按照模板格式，不得随意变更
3. **逻辑清晰**：剧情点之间有明确的因果关系
4. **分集合理**：考虑单集容量和节奏控制
5. **钩子准确**：情绪钩子类型标注必须准确

## 注意事项

- 每批处理6章小说内容
- 剧情点数量根据冲突密度确定，无固定数量
- 重点提取能引发情绪反应的关键情节
- 删除纯铺垫内容，保留核心冲突
- 状态管理确保不重复使用剧情点
- 分集标注要考虑跨集连贯性
